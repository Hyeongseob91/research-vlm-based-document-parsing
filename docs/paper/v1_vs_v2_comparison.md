## 4. 분석

### 4.1 v2가 개선된 부분

1. **Skip Rate 0%**: v2는 1,355페이지 전체를 처리. v1은 79페이지(5.8%) 실패
   - 원인 추정: rank 32로 모델 용량이 커져 다양한 문서 유형에 대한 robustness 향상
   - 단, v2는 TP=2로 서빙(더 많은 GPU 메모리)했고, v1은 TP=1이었으므로 OOM 차이일 가능성 있음

2. **Formula CDM F1 개선 (0.884 → 0.892)**: rank 증가의 원래 목적이 수식 인식 개선이었고, 소폭 달성

3. **Text page_avg NED 개선 (0.304 → 0.300)**: 미세하지만 개선

### 4.2 v2가 퇴보한 부분

1. **Table TEDS 큰 하락 (0.649 → 0.610, -3.9pp)**: 가장 문제적인 결과
   - v1에서 teacher(0.523)를 크게 초월했던 강점이 약화됨
   - TEDS-S도 동반 하락 (0.732 → 0.695)

2. **Text sample_avg NED 하락 (0.288 → 0.306, +1.8pp)**: 개별 텍스트 블록 수준에서 퇴보

### 4.3 원인 분석

**v2 퇴보의 주요 원인: Overfitting**

| 근거 | 설명 |
|------|------|
| Eval loss 추이 | Step 1000 (epoch 3)에서 best (0.091), 이후 상승 (0.096) |
| v1 best도 epoch 3 | v1은 epoch 3에서 끝나서 자연스럽게 best에 가까움 |
| Table 성능 하락 | Overfitting은 학습 데이터에 과적합 → 벤치마크 성능 하락 |
| Rank 32 + Epoch 5 | 파라미터 4배 + epoch 증가 = overfitting 위험 증가 |

**핵심**: rank를 32로 올린 것은 표현력을 높였지만, epoch 5는 과했음. **Best checkpoint은 step 1000 (epoch 3)**이었으나, 최종 checkpoint (step 1670)으로 merge했기 때문에 overfitting된 모델을 평가한 것.

### 4.4 Skip Rate 0%에 대한 주의

v2의 skip rate 0%는 인상적이지만, **v1과 서빙 조건이 달랐음**:
- v1: TP=1 (GPU 1장, 약 98GB) → 큰 이미지에서 OOM 발생 가능
- v2: TP=2 (GPU 2장, 약 196GB) → OOM 여유 충분

따라서 skip rate 개선이 모델 자체의 개선인지, 서빙 환경 차이인지 분리할 수 없음.

## 5. 검토 의견 및 권장 사항

### 5.1 결론

**v2 (rank 32, epoch 5, 최종 checkpoint)는 v1 대비 전반적으로 퇴보했습니다.**

- 핵심 강점이었던 Table TEDS가 -3.9pp 하락
- 수식 개선은 +0.8pp로 미미
- Skip rate 개선은 서빙 환경 차이 가능성

### 5.2 권장 사항

| 우선순위 | 액션 | 기대 효과 |
|---------|------|----------|
| **1 (즉시)** | v2의 **best checkpoint (step 1000)으로 재평가** | Epoch 3 시점의 모델 → overfitting 이전, v1 대비 순수하게 rank 효과만 비교 가능 |
| 2 | v1을 TP=2로 재추론하여 skip rate 동등 비교 | Skip rate 차이가 모델 vs 환경 중 어디서 오는지 확인 |
| 3 | rank 32 + epoch 3 조합 학습 (v2.1) | rank 증가의 이점만 취하고 overfitting 회피 |
| 4 | Early stopping 도입 | Eval loss 기반 자동 중단으로 overfitting 방지 |

### 5.3 현재 최선의 모델

**현 시점에서는 v1 (rank 8, epoch 3)이 최선의 모델입니다.**

단, v2의 best checkpoint (step 1000) 평가 결과에 따라 바뀔 수 있습니다. 이 checkpoint은 v1과 동일한 epoch 수에서의 모델이지만 rank가 4배 높으므로, overfitting 없이 rank 증가의 순수 효과를 확인할 수 있습니다.
